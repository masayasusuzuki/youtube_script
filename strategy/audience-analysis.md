# 視聴者分析（AI駆動開発関連動画のコメント欄より）

## 視聴者層のレベル感

### 非エンジニア・初心者層
- 「非エンジニアです」と自称する人が複数
- 「バイブコーディング」「仕様型駆動コーディング」などの用語を使い始めている
- 社内向けシステムを自分で作ろうとしている
- ディレクトリ構成やドキュメント管理で悩んでいる

### 中級者層
- Cursor、Claude Code、GitHub Issue連携など実践している
- `.mdc`ファイルやルール設定を試行錯誤中
- 「スパゲッティコードにならないように」という課題意識

### 懐疑的・限界を感じている層
- 「AIで開発してますが、限界を感じてます」
- 「事務の方ができるとは到底思えない」
- 「HPレベルが関の山」

## 視聴者が気になっていること（ニーズ）

### 1. 基礎知識・前提スキル
- 「結局、自力である程度書けないとダメ？」
- 「必要な事前知識 = 中堅レベルエンジニアになること？」
- 「読める」レベルの知識パッケージが欲しい

### 2. セキュリティ
- 「個人開発で知っておくべきセキュリティ要件は？」
- 「機密情報漏洩の観点で導入できない」

### 3. 実践的なワークフロー
- コードがスパゲッティにならない方法
- タスク分割のやり方
- ドキュメント管理（docs/、plan.md、changelog）
- GitHub Issue活用

### 4. ツールの使い分け・設定
- User rules と Project rules の違い
- `.mdc`ファイルの書き方
- Claude Code の設定・運用方法

### 5. 具体的なユースケース
- 社内向けシステム開発
- ホームページレベルの開発
- ビジネスアプリ開発

## コメントから見える「刺さるコンテンツ」

### 初心者が求めているもの
- 「読める」レベルになるための最低限の知識
- 挫折ポイントの解説
- 段階的なステップアップ

### 実践者が求めているもの
- 実際の運用ルール・設定の公開
- ワークフローの具体例
- 失敗談・つまずきポイント

## 1本目の候補

### A案：概念説明から入る
**「AI駆動開発って何？非エンジニアでも本当にできるの？」**

メリット：
- 期待値調整＋可能性を示す動画が入口として強い
- マサの実体験がそのまま説得力になる
- チャンネルの立ち位置を明確にできる

### B案：実践から入る
**「Claude Code導入ガイド」**

メリット：
- 検索流入が見込める
- すぐに手を動かしたい人に刺さる
- チャンネル紹介は概要欄やショートでカバー可能
